üñ®Ô∏è Printing in Python

Python's print() function is versatile and supports many features:

1. Basic Printing

2. Printing Multiple Items

print("A", "B", "C")  # Output: A B C

3. Custom Separator (sep)

print("A", "B", "C", sep="-")  # Output: A-B-C

4. Custom End (end)

print("Hello", end=" ")
print("World!")  # Output: Hello World!

5. Formatted Strings (f-strings)

score = 99
print(f"Your score is {score}!")

üìä Day 2: Variables and Data Types

Day 2 Variables & Data Types Beginner

üîë Variables: The Building Blocks

  • üì¶ Variable: A named location in memory to store data
  • ‚û°Ô∏è Assignment: Use = to assign a value
  • üîÑ Dynamic Typing: Python variables can change type
  • üî† Case Sensitive: score and Score are different

üìù Naming Conventions (PEP 8)

  • Use snake_case: user_name, total_score
  • Start with a letter or underscore (_)
  • Cannot start with a number
  • Only letters, numbers, underscores
  • Case-sensitive

üìä Data Types in Python

Type Example Description
int 42 Whole numbers
float 3.14 Decimal numbers
str "Hello" Text data
bool True True/False

üîÑ Type Conversion (Casting)

  • int(): Convert to integer
  • float(): Convert to float
  • str(): Convert to string
  • bool(): Convert to boolean

üìú Examples

1. Declaring variables of all types

age = 21           # int
price = 19.99      # float
name = "Alice"     # str

2. Type conversion

print(num_int, num_float)
print(type(num_int), type(num_float))
üö© Common Mistakes:
  • Forgetting quotes for strings: name = Alice (error)
  • Mixing types: print("Age: " + 21) (error, must convert 21 to string)
  • Using reserved words as variable names: class = "Math" (error)

Operators are special symbols that perform operations on variables and values. They are the backbone of calculations, logic, and data manipulation in Python.

üéØ Type üìù Description üîß Examples
üßÆ Arithmetic Math operations +, -, *, /, %, **, //
‚öñÔ∏è Comparison Compare values ==, !=, >, <, >=, <=
üß† Logical Boolean logic and, or, not
‚ÜîÔ∏è Assignment Assign values =, +=, -=, *=, /=, %=, //=, **=
üß© Bitwise Bit-level operations &, |, ^, ~, <<, >>

üèÖ Full Python Operator Precedence Table

üìä Operators are evaluated in the following order (highest to lowest):
üèÜ Precedence üîß Operator(s) üìù Description
1 () Parentheses (grouping)
2 x[index], x.attribute, x(...) Indexing, slicing, attribute reference, function call
3 ** Exponentiation
4 +x, -x, ~x Unary plus, minus, bitwise NOT
5 *, /, //, % Multiplication, division, floor division, modulus
6 +, - Addition, subtraction
7 <<, >> Bitwise shift left, right

12 not Logical NOT

15 if - else Conditional expression

17 =, +=, -=, etc. Assignment
üí° Pro Tip: When in doubt, use parentheses () to make your code clear and avoid precedence mistakes!

üìú Examples

1. Arithmetic Operators

a = 10
b = 3
print("Add:", a + b)
print("Subtract:", a - b)
print("Multiply:", a * b)
print("Divide:", a / b)
print("Modulus:", a % b)
print("Exponent:", a ** b)
print("Floor Div:", a // b)

2. Comparison Operators

x = 5
y = 10
print(x == y)  # False
print(x != y)  # True
print(x > y)   # False
print(x <= y)  # True
üö© Common Mistakes:
  • Using = instead of == in comparisons
  • Integer division confusion: 5 / 2 gives 2.5, 5 // 2 gives 2
  • Operator precedence errors: 2 + 3 * 4 is 14, not 20
